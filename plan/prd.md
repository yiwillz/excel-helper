# 产品需求文档 (PRD)：DataCopilot - 离线桌面 Excel AI 助手

## 1. 产品概述 (Product Overview)
* **产品名称：** DataCopilot (便携版)
* **产品定位：** 一款**零环境依赖、完全离线**的轻量级 Windows 桌面数据处理工具。
* **核心价值：** 让不懂代码和复杂 Excel 公式的用户，通过自然语言对话，自动完成表格的筛选、计算、聚合等复杂操作。数据全程留在本地，绝对保护隐私。
* **目标部署环境：** 最低配置要求为 8GB 内存、无独立显卡的轻薄本（以 Surface Pro 8 i5-1135G7 为基准）。

## 2. 系统架构与交付形态 (Architecture & Delivery)
本产品采用“便携式免安装”架构，最终交付给用户的是一个约 **1.5 GB 的 ZIP 压缩包**。解压后即用，无需配置任何系统环境变量。

* **前端/胶水层 (UI & Control)：** Python + Tkinter（极简原生界面，极低内存消耗），最后由 `PyInstaller` 编译为单一的 `DataCopilot.exe`。
* **数据引擎 (Data Engine)：** `DuckDB` (内嵌于 Python)，负责将 Excel 加载为内存数据库，并高速执行 SQL 操作导出新表。
* **大模型引擎 (LLM Engine)：** 官方预编译版 `llama-server.exe`。
* **AI 模型 (Model)：** `Qwen2.5-Coder-1.5B-Instruct-GGUF` (Q4_K_M 量化版本，约 1.1GB)，专精 SQL 生成和中文理解。

## 3. 核心功能流 (Core User Flow)

### 3.1 启动与初始化 (Initialization)
* **F1. 静默拉起服务：** 用户双击 `DataCopilot.exe` 后，程序自动在后台隐蔽启动 `llama-server.exe` 并加载模型文件。
* **F2. 内存锁死机制：** 启动参数强制设定上下文窗口大小（如 `-c 1024`），确保系统总内存占用峰值死死卡在 **2GB** 以内，防止系统崩溃。

### 3.2 文件读取与对话交互 (Interaction)
* **F3. 文件加载：** 提供一个简单的界面或弹窗，让用户选择要处理的 `.xlsx` 或 `.csv` 文件。
* **F4. 表头提取：** 后台自动读取表格的第一行（列名）及数据类型，拼接成 System Prompt（“你是一个数据分析师，当前表包含以下列：...”）。
* **F5. 自然语言输入：** 提供文本框供用户输入操作指令（如：“统计各部门的平均薪资”）。

### 3.3 意图识别与 AI 处理 (AI Processing)
* **F6. Text-to-SQL 转换：** 模型接收指令与表头，直接输出且仅输出标准的 DuckDB SQL 查询语句。
* **F7. 强制澄清机制 (Human-in-the-loop)：** * **规则设定：** 若用户指令模糊（如“整理一下这个表”、“分析数据”），模型被设定为**拒绝生成 SQL**。
    * **执行动作：** 模型需输出自然语言反问用户（如：“请问您需要按哪个维度整理？需要提取哪些特定的列？”），直至指令明确。

### 3.4 任务执行与容错 (Execution & Recovery)
* **F8. SQL 本地执行：** Python 提取模型输出的 SQL 并在内嵌的 DuckDB 中极速运行。
* **F9. 自动纠错 (Self-Correction)：** * 若 SQL 执行报错（如列名拼写错误），Python 会捕获报错信息。
    * 系统在后台自动将“原 SQL + 报错信息”发回给大模型要求修改，最多静默重试 2 次。
* **F10. 结果导出：** 执行成功后，自动在源文件同级目录下生成 `Result_时间戳.xlsx`，并在界面提示用户“处理完成”。

## 4. 边界条件与限制 (Out of Scope - 预期管理)
为保证 1.5B 模型在极低算力下的稳定性，以下功能属于**绝对禁区**，不支持且不开发：
1.  **不支持图表生成：** 无法绘制折线图、饼图等可视化内容。
2.  **不支持格式排版：** 无法修改单元格颜色、字体加粗、合并单元格等视觉样式。
3.  **不支持跨文件复杂操作：** 本期 V1.0 仅支持单表/单文件的数据处理。
4.  **不支持主观业务分析：** 模型不具备常识，无法回答“为什么上个月销量下滑”等主观洞察问题。

## 5. 性能与资源指标 (Performance Metrics)
* **存储空间占用：** < 2 GB（含模型、引擎、程序）。
* **运行内存占用：** 峰值 < 2 GB。
* **响应时间：** AI 思考并生成 SQL 的时间需控制在 **3 - 8 秒**之间（基于 i5-1135G7 CPU 纯核算力预期）。
* **离线合规：** 断开 Wi-Fi 必须 100% 正常运行。
